# Security Policy

## Supported Versions

Security fixes are prioritized for the default branch (`main`) and the latest production deployment.
Older snapshots and experimental branches may not receive patches.

## Reporting a Vulnerability

Please do not create a public issue for undisclosed vulnerabilities.

Report privately by email:

- `mail@earlysalty.com`

Include, if possible:

- Affected component or file path
- Reproduction steps or proof-of-concept
- Impact assessment (what can be accessed, changed, or executed)
- Suggested remediation, if known

## Response Process

- Initial acknowledgement target: within 72 hours
- Triage and severity classification target: within 7 days
- Fix timeline target:
  - Critical: as soon as possible, usually within 7 days
  - High: usually within 14 days
  - Medium/Low: best-effort, prioritized with normal release work

If a report is accepted, we will coordinate disclosure timing with the reporter.

## Disclosure and Credits

- We prefer coordinated disclosure after a patch is available.
- Reporter credit can be included in release notes unless anonymity is requested.
